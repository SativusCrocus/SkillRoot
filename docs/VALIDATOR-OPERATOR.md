# Validator Operator Guide

This guide walks a new operator through running a SkillRoot v0 validator end-to-end on Base Sepolia. It assumes:

- You've been granted or purchased ≥1,000 SKR on Base Sepolia
- You have a wallet whose private key you control
- You run macOS or Linux with a reasonable uplink
- You're comfortable on the command line

If any of these are false, stop here and ask in the operator channel (see [Communication](#communication)).

> **v0 disclaimer:** SkillRoot v0 is an unaudited testnet prototype. Bonded SKR is slashable. Read `docs/threat-model.md` before proceeding. Do not run with real assets.

## 1. Install the toolchain

```bash
git clone https://github.com/SativusCrocus/SkillRoot.git
cd SkillRoot
./scripts/setup.sh
```

`setup.sh` is idempotent. It installs Foundry, Node 20, pnpm 9, Rust, Circom 2, and snarkjs 0.7. You can re-run it anytime to verify or upgrade your environment.

Sanity check:

```bash
forge --version
node --version     # v20.x or higher
pnpm --version     # 9.x or higher
snarkjs --version
circom --version
```

## 2. Build the CLI

```bash
cd cli
pnpm install
pnpm build
pnpm link --global
skr --help
```

You should see the six subcommands: `challenges`, `solve`, `submit`, `query`, `stake`, `validate`.

## 3. Configure the deployment target

The CLI reads a deployment JSON that lists the contract addresses for the network you're targeting. For Base Sepolia, that file is at `deployments/base-sepolia.json` and is generated by the protocol maintainer after each redeploy.

```bash
export PRIVATE_KEY=0x...                                  # your validator key
export SKR_CHAIN_ID=84532                                 # Base Sepolia
export SKR_RPC_URL=https://sepolia.base.org               # or your own RPC
export SKR_DEPLOYMENT="$(pwd)/deployments/base-sepolia.json"
```

Persist these in your shell profile (`~/.zshrc` or `~/.bashrc`) or a local `.envrc` if you use direnv. **Never commit your private key.**

## 4. Fund your validator address

You need two things in your wallet:

1. **Base Sepolia ETH** for gas. Grab it from [Alchemy](https://www.alchemy.com/faucets/base-sepolia), [QuickNode](https://faucet.quicknode.com/base/sepolia), or the Coinbase faucet. ~0.01 ETH is plenty.
2. **SKR** for bonding. For the bootstrap phase, the maintainer grants each new operator 5,000 SKR from the Governance treasury. DM your address in the operator channel to receive the grant.

Verify with:

```bash
cast balance $(cast wallet address --private-key $PRIVATE_KEY) --rpc-url $SKR_RPC_URL --ether
cast call $(jq -r '.contracts.SKRToken' $SKR_DEPLOYMENT) \
  "balanceOf(address)(uint256)" \
  $(cast wallet address --private-key $PRIVATE_KEY) \
  --rpc-url $SKR_RPC_URL
```

## 5. Bond as a validator

Minimum stake is **1,000 SKR**. You can bond more — your committee-draw probability is proportional to your stake.

```bash
skr stake 1000
```

This does two transactions:
1. `SKRToken.approve(StakingVault, 1000e18)`
2. `StakingVault.bond(1000e18)`

After it succeeds, check your active stake:

```bash
cast call $(jq -r '.contracts.StakingVault' $SKR_DEPLOYMENT) \
  "stakeOf(address)(uint256)" \
  $(cast wallet address --private-key $PRIVATE_KEY) \
  --rpc-url $SKR_RPC_URL
```

## 6. Run the validator daemon

```bash
skr validate
```

The daemon:
1. Polls `AttestationEngine` for `CommitteeDrawn` events every 4 seconds
2. Checks whether your address is in each committee
3. For membership hits, it votes (see Voting policy below)
4. Tracks processed claims in-memory so you don't double-vote

Recommended: run it under a process manager so it restarts on reboot.

### systemd (Linux)

```ini
# /etc/systemd/system/skr-validator.service
[Unit]
Description=SkillRoot v0 validator
After=network.target

[Service]
Type=simple
User=skr
WorkingDirectory=/home/skr/skillroot
EnvironmentFile=/home/skr/.skr/env
ExecStart=/home/skr/.local/bin/skr validate
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now skr-validator
journalctl -u skr-validator -f
```

### tmux / screen (quick start)

```bash

# Ctrl-b d to detach; tmux a -t skr to reattach
```

## 7. Voting policy

v0's validator daemon votes **YES** on every claim where it is drawn into committee. This is safe because:

1. `AttestationEngine.submitClaim` already runs the real Groth16 verifier on-chain. A claim that reached the committee stage has a mathematically valid proof — there is no "should I verify?" question to answer.
2. The D2 binding (`keccak256(abi.encode(msg.sender, challengeId))`) prevents front-running, so the committee-observed proof is necessarily bound to the correct claimant.
3. Silent voters are slashed 1% (liveness). Voting NO on a valid proof is slashed 5% (equivocation). Voting YES is the dominant strategy.

If you want to double-check off-chain (for extra paranoia), run:

```bash
skr validate --vkey ./circuits/math/build/verification_key.json
```

The flag exists for future circuits where the on-chain verifier might be swapped, but is a no-op in v0.

**Do not use `--skip-verify` in production.** It's a testing flag that bypasses even the minimal sanity checks.

## 8. Monitoring your validator

### Health-check commands

```bash
# Are you still bonded above the minimum?
cast call $(jq -r '.contracts.StakingVault' $SKR_DEPLOYMENT) \
  "stakeOf(address)(uint256)" \
  $(cast wallet address --private-key $PRIVATE_KEY) \
  --rpc-url $SKR_RPC_URL

# How many validators are currently active?
cast call $(jq -r '.contracts.StakingVault' $SKR_DEPLOYMENT) \
  "validatorCount()(uint256)" \
  --rpc-url $SKR_RPC_URL

# Your decayed attestation scores (0 unless you also make claims)
skr query $(cast wallet address --private-key $PRIVATE_KEY)
```

### Watch for slashing

`StakingVault` emits a `Slashed(address,uint256,uint256)` event every time a validator is slashed. Subscribe via:

```bash
cast logs \
  --rpc-url $SKR_RPC_URL \
  --address $(jq -r '.contracts.StakingVault' $SKR_DEPLOYMENT) \
  "Slashed(address,uint256,uint256)" \
  --from-block latest
```

If your address appears in a slash event, the daemon missed a vote or voted the wrong direction. Check `journalctl -u skr-validator` for the corresponding `poll error` or `vote failed` line.

### Dashboard (optional)

The v0 dApp is live at [`https://app-nine-rho-70.vercel.app`](https://app-nine-rho-70.vercel.app). It currently exposes `/`, `/submit`, and `/me`. A `/validators` dashboard listing bonded addresses, stake, and recent committee hits is a planned v1 addition (see [`ROADMAP.md`](ROADMAP.md)).

## 9. Unbonding

When you want to exit:

```bash
# Request partial or full unbond
cast send $(jq -r '.contracts.StakingVault' $SKR_DEPLOYMENT) \
  "requestUnbond(uint256)" \
  1000000000000000000000 \
  --rpc-url $SKR_RPC_URL --private-key $PRIVATE_KEY

# Wait 14 days, then withdraw
cast send $(jq -r '.contracts.StakingVault' $SKR_DEPLOYMENT) \
  "withdraw()" \
  --rpc-url $SKR_RPC_URL --private-key $PRIVATE_KEY
```

Partial unbonds must leave ≥1,000 SKR still staked or you must fully exit.

Stop the daemon before or immediately after requesting a full unbond — once your stake is below `MIN_STAKE`, you'll be removed from the validator array and can no longer be drawn into committees.

## 11. Known v0 rough edges

Coming from `docs/threat-model.md`:

1. **Sequencer-trust randomness** — committee draws use `blockhash(submissionBlock + 4)`. Mitigated on Base Sepolia; will move to VRF before mainnet.
2. **Single-party trusted setup** — Phase 2 was run by the v0 deployer alone. Pre-mainnet, a ≥3-party ceremony will re-deploy the verifier.
3. **No timelock on governance** — proposals execute the instant voting closes. The bootstrap multi-sig adds a manual gate.
4. **One circuit only** — modexp is the sole challenge. Three more (`algo`, `fv`, `sec`) ship in v1.
5. **No fee market** — v0 validators earn nothing directly. Weeks 7–8 cover your costs via grants; the fee market lands in v1.

All of these are documented in `docs/ROADMAP.md`.

## Appendix: troubleshooting

**`skr validate` exits immediately with "poll error"**
→ Check `SKR_RPC_URL` is reachable: `cast chain-id --rpc-url $SKR_RPC_URL`.

**`vote failed: NotCommitteeMember`**
→ Race: the daemon saw a `CommitteeDrawn` event but your address isn't in that particular committee. Harmless — it only processes committees where you're a member. If you see this *every* time, check that the wallet address matches your bonded address.

**`bond failed: BelowMinStake`**
→ You tried to bond less than 1,000 SKR, or a previous partial unbond dropped you below the floor. Bond at least enough to return above 1,000 total.

**`approve` tx pending forever**
→ Base Sepolia occasionally has slow blocks. Retry with a higher gas price via `--gas-price`. Check a block explorer like https://sepolia.basescan.org.

**Validator address not receiving grants**
→ The grant transaction is sent by the maintainer from the governance treasury. Double-check the address you shared matches exactly the one you're bonding from.
